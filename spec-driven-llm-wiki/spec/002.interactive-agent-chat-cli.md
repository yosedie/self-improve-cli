# Spec 002: Interactive Agent Chat CLI

## Task/Prompt

User can run `init` but cannot start an interactive coding agent yet. Implement a lightweight chat command so the CLI can accept prompts, call an OpenAI-compatible model, run local tools, and return answers or code edits.

## Tujuan

Tambahkan mode agent CLI yang bisa dipakai langsung:

- `sicli chat` untuk interactive REPL.
- `sicli chat "prompt"` untuk one-shot task.
- Config provider/model/API key env di `.selfimprove/config.json`.
- OpenAI-compatible Chat Completions support via native `fetch`.
- Tool calling loop for `read_file`, `search`, `run_command`, and `edit_file`.
- Respect tool policy from active JSON profile.
- Stay zero-dependency and low memory.

## Kenapa Tujuan Ini

MVP sebelumnya hanya fondasi profile/growth/tools. Agar terasa sebagai coding agent, user butuh loop prompt -> model -> tools -> hasil. Implementasi harus tetap kecil: no TUI heavy, no watcher, no indexer, no dependency.

## Logic Changes

- Add config state file `.selfimprove/config.json` with safe defaults.
- Add `config get|set|show` commands.
- Add provider client that POSTs to `/chat/completions` using `fetch`.
- Add tool schema definitions for provider function calling.
- Add agent loop that:
  - builds system prompt from active profile,
  - sends user prompt,
  - executes requested tool calls,
  - appends tool results,
  - stops when assistant returns final text or max turns hit.
- Add interactive `readline` REPL.
- Add `edit_file` tool with exact single replacement to avoid broad rewrites.
- Enforce profile tool policies: `allow`, `deny`, `ask`.

## Pseudocode

```text
config init:
  create .selfimprove/config.json if absent
  defaults:
    base_url env SICLI_BASE_URL or https://api.openai.com/v1
    api_key_env OPENAI_API_KEY
    model env SICLI_MODEL or gpt-4o-mini
    max_tool_turns 8

chat:
  if no prompt:
    start readline loop
  else:
    run one shot

agent run:
  load active profile
  load config
  build messages: system + history + user
  repeat max_tool_turns:
    response = provider.chat(messages, tools)
    append assistant message
    if no tool_calls: print final and stop
    for each tool_call:
      check policy
      if ask and no approval: return policy error
      run local tool
      append tool result
  return max turns error

edit_file:
  read file
  require old_text occurs exactly once
  replace old_text with new_text
  write file
```
