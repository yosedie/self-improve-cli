# Spec 005: Tool Approval and File Write Fixes

## Task/Prompt

User found interactive agent bugs: approval prompt echoes typed `y` twice, model repeatedly tries shell redirection through `run_command`, and simple file creation task stops after max tool turns instead of creating the file. User expects OS/tool feedback and robust behavior.

## Tujuan

Fix interactive tool loop quality:

- Use one readline instance for chat and approvals to avoid duplicate echo.
- Add `write_file` tool for creating/overwriting text files without shell redirection.
- Teach system prompt that `run_command` uses `shell: false`; no pipes/redirection/heredocs.
- Add workspace OS/cwd feedback in system prompt.
- Print concise tool progress feedback during interactive sessions.
- Strip `<think>...</think>` blocks from final user-visible output.

## Kenapa Tujuan Ini

The agent should create files with direct tools, not fight platform shells. Approval UX must be trustworthy. User-visible feedback helps diagnose tool activity without requiring trace mode.
