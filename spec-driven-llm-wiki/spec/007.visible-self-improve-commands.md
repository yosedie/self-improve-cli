# Spec 007: Visible Self-Improve Commands

## Task/Prompt

User cannot see where self-improvement happens and asks for a concrete example or fix. Existing system logs failures and can patch profiles, but the UX is hidden behind `improve` and automatic chat failure logging.

## Tujuan

Make self-improvement visible and testable:

- Add `self-improve status` command.
- Add `self-improve learn <message> [--apply]` command.
- Add `self-improve demo [--apply]` command with a known tool-misuse lesson.
- Add `self-improve background-run` command.
- Add chat `/self-improve` and `/self-improve demo` slash commands.
- Record chat task traces into `.selfimprove/traces.jsonl`.
- Run non-blocking background review after tasks when enabled.
- Show growth mode, auto-apply state, recent traces, recent events, recent patches, and next action.
- Keep safety gates: patches still validate through growth policy.

## Kenapa Tujuan Ini

Self-improve is not useful if the user cannot observe, test, or understand it. A deterministic demo helps prove the loop without requiring provider API calls.

## Pseudocode

```text
selfImproveLearn(root, message, apply):
  active = load profile
  event = append event
  suggestion = suggestPatchFromEvent(event)
  gate = evaluatePatch(active, suggestion.patch, { manual: apply })
  if gate.allowed and (gate.auto or apply): apply patch
  append patch audit
  return event/suggestion/audit/status

self-improve status:
  print growth, events count, patches count, recent patch, overlay path

self-improve demo:
  message = "agent used run_command shell redirection for file creation"
  run learn(message)
```
