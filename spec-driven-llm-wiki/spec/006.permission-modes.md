# Spec 006: Permission Modes

## Task/Prompt

User wants four permission modes for agent tool execution:

1. secure: always ask.
2. partial secure: allow actions reversible by git; ask if not reversible.
3. AI-reviewed: a clean-context reviewer model approves safety; ask user if reviewer rejects or cannot run.
4. auto approve: autopilot until completion.

## Tujuan

Add configurable permission policy above profile tool policy:

- `secure`: ask before every tool call.
- `partial_secure`: allow read/search; allow file writes/edits only when git-reversible; ask for run commands and non-reversible writes.
- `ai_reviewed`: allow read/search; ask a clean-context reviewer model for action tools; ask user if review denies/errors.
- `auto_approve`: allow all profile-permitted tools.
- Add CLI and chat controls: `permissions` command and `/permissions` slash command.
- Keep profile `deny` absolute.

## Kenapa Tujuan Ini

User needs security levels matching different workflows: safe manual mode, practical git-backed automation, paid AI reviewer mode, and full autopilot.

## Pseudocode

```text
ensureAllowed(root, profile, config, tool, args):
  if profile policy deny: deny
  if permission_mode auto_approve: allow
  if permission_mode secure: ask
  if permission_mode partial_secure:
    if read/search: allow
    if write/edit git reversible: allow
    else ask
  if permission_mode ai_reviewed:
    if read/search: allow
    reviewer = call model with clean messages and no tools
    if reviewer approved: allow
    else ask user with reason

isGitReversibleFileAction:
  if not git repo: false
  if target missing: true
  if target tracked and clean: true
  else false
```
