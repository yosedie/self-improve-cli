# Spec 008: Escape Cancellation

## Task/Prompt

User wants to cancel running agent tasks by pressing the `ESCAPE` key. Currently, the CLI only handles `Ctrl+C` (interrupt) to exit the entire process. The agent loop must be interruptible while remaining in the REPL.

## Tujuan

Implement `ESCAPE` key interruption:

- Use `AbortController` to interrupt async agent loops and tools.
- Capture `ESCAPE` key during task execution.
- Maintain REPL functionality after task cancellation.
- Tool cleanup (e.g., kill child processes) on abort.

## Kenapa Tujuan Ini

Long-running agent tasks (like infinite loops or slow commands) can currently only be stopped by killing the entire CLI process. `ESCAPE` allows stopping the *task* without losing configuration or history.

## Logic Changes

- `startChat` sets up `readline.emitKeypressEvents(process.stdin)`.
- When task starts:
  - `process.stdin.setRawMode(true)`
  - Listen for `keypress` and check `key.name === 'escape'`.
  - Abort signal triggers task cancellation.
- When task ends:
  - `process.stdin.setRawMode(false)`
  - Stop listening to `keypress`.
- Pass `AbortSignal` through `runAgentTask`, `chatCompletion` (fetch), and `runCommandTool` (spawn).

## Code Changes

- Modify `src/agent.js` (main loop and keypress handling).
- Modify `src/provider.js` (fetch signal).
- Modify `src/tools.js` (runCommandTool signal).
- Update Wiki/Log.

## Pseudocode

```text
controller = new AbortController()
stdin.setRawMode(true)
stdin.on('keypress', (str, key) => {
  if (key.name === 'escape') controller.abort()
})
runAgentTask(root, prompt, { signal: controller.signal })
// ...
// catch signal.aborted: print "Cancelled"
```
