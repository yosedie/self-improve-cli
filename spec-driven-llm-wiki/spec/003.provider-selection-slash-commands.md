# Spec 003: Provider Selection Slash Commands

## Task/Prompt

User started interactive chat and typed `/connect` and `/models`, but both were sent to the model and failed because `OPENAI_API_KEY` was missing. Implement local slash commands so `/connect` selects provider and `/models` selects model before any provider API call. Add built-in MiniMax Coding Plan and Z.AI Coding Plan presets from user-provided docs.

## Tujuan

Buat chat REPL bisa konfigurasi provider/model secara lokal:

- `/connect` lists and selects provider without requiring API key.
- `/connect minimax` and `/connect zai` set provider directly.
- `/models` lists and selects model for current provider without requiring API key.
- `/models <model>` sets model directly.
- Include MiniMax Coding Plan OpenAI-compatible preset:
  - Base URL `https://api.minimax.io/v1`
  - API key env `MINIMAX_API_KEY`
  - Models `MiniMax-M2.7`, `MiniMax-M2.7-highspeed`
- Include Z.AI Coding Plan OpenAI-compatible preset:
  - Base URL `https://api.z.ai/api/coding/paas/v4`
  - API key env `ZAI_API_KEY`
  - Models `GLM-5.1`, `GLM-5`, `GLM-5-Turbo`, `GLM-4.7`, `GLM-4.5-air`
- Do not store raw API keys.

## Kenapa Tujuan Ini

User needs first-run UX that does not require manual config commands or fail on local slash commands. Provider choice should be discoverable inside chat.
