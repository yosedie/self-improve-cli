# Spec 010: MCP Chat/CLI Commands

## Task/Prompt

Add `/mcp` chat slash commands and `sicli mcp` CLI subcommands to manage MCP server connections. Users must be able to add, remove, list, and reload MCP servers from both the interactive chat and the command line. Config persisted to `.selfimprove/mcp.json`. Depends on spec 009 (MCP protocol client + tool bridge).

## Tujuan

Menyediakan antarmuka terpadu (chat slash commands + CLI subcommands) untuk mengelola koneksi MCP server:

- Tambah server MCP (stdiO command atau remote URL) dan langsung konek.
- Hapus server MCP, putuskan koneksi, dan bersihkan dari config.
- Tampilkan daftar server yang terhubung beserta status dan jumlah tool.
- Reload semua server dari config file.
- Semua perubahan disimpan ke `.selfimprove/mcp.json`.

## Kenapa Tujuan Ini

Pengguna perlu cara mudah mengelola MCP server tanpa edit file JSON manual. Tanpa command ini, setiap perubahan konfigurasi MCP membutuhkan edit file langsung dan restart CLI. Dengan slash commands dan CLI subcommands, workflow menjadi:

1. `/mcp add filesystem npx @modelcontextprotocol/server-filesystem /tmp` langsung konek.
2. `/mcp list` tunjukkan status.
3. `sicli mcp list` untuk scripting/CI.
4. `/mcp reload` untuk reconnect setelah edit manual config.

Integrasi dengan `handleSlashCommand` dan CLI dispatcher yang sudah ada meminimalkan kode baru.

### Existing Patterns

- **handleSlashCommand** dispatches `/command arg` by splitting on whitespace, delegating to handler functions like `handleConnectCommand`, `handleSwarmCommand`.
- **CLI main()** uses `parseFlags(args)` for `--flag value` parsing, then dispatches by `command` string with `if/else if` chains.
- **state.js** uses `statePath(root, filename)` + `readJson`/`writeJson` for all persisted state files.
- Config files use `{ recursive: true }` mkdir before write.
- All async, all plain JS, no external dependencies.
- Handler functions return `true` to keep chat loop alive, `false` to exit.

### Config Format

`.selfimprove/mcp.json`:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-filesystem", "/tmp"],
      "env": {}
    },
    "remote-api": {
      "url": "https://example.com/mcp",
      "headers": { "Authorization": "Bearer ..." }
    }
  }
}
```

## Logic Changes

1. **State layer** — Add `loadMCPConfig(root)` and `saveMCPConfig(root, config)` to `src/state.js`. These follow the same `statePath` + `readJson`/`writeJson` pattern as existing state functions.

2. **Chat command handler** — Add `handleMCPCommand(root, arg, mcpBridge)` to `src/agent.js`. Parses subcommand (`add`, `remove`, `list`, `reload`) and delegates.

3. **CLI subcommand** — Add `mcp` dispatch block to `bin/self-improve-cli.js` `main()`. Parses subcommand and flags, calls same state functions + MCP bridge.

4. **MCP bridge dependency** — The actual connect/disconnect/tool-count logic delegates to the MCP bridge from spec 009. This spec only covers the command/UX layer. If MCP bridge is unavailable, commands fail gracefully with a clear message.

## Pseudocode

```text
// === src/state.js additions ===

MCP_CONFIG = 'mcp.json'

loadMCPConfig(root):
  fp = statePath(root, MCP_CONFIG)
  return readJson(fp, { mcpServers: {} })

saveMCPConfig(root, config):
  fp = statePath(root, MCP_CONFIG)
  writeJson(fp, config)


// === src/agent.js: handleMCPCommand ===

handleMCPCommand(root, arg, mcpBridge):
  [subcommand, ...rest] = arg.split(/\s+/)

  if subcommand == 'add':
    return handleMCPAdd(root, rest, mcpBridge)

  if subcommand == 'remove':
    return handleMCPRemove(root, rest, mcpBridge)

  if subcommand == 'list':
    return handleMCPList(root, mcpBridge)

  if subcommand == 'reload':
    return handleMCPReload(root, mcpBridge)

  print usage:
    "/mcp add <name> <command> [args...] [--env KEY=VAL]"
    "/mcp remove <name>"
    "/mcp list"
    "/mcp reload"
  return true


handleMCPAdd(root, tokens, mcpBridge):
  if tokens.length < 2:
    print "usage: /mcp add <name> <command> [args...] [--env KEY=VAL]"
    return true

  name = tokens[0]
  command = tokens[1]
  args = []
  env = {}

  // parse remaining tokens: collect args until --env, then KEY=VAL pairs
  i = 2
  while i < tokens.length:
    if tokens[i] == '--env':
      i++
      while i < tokens.length and tokens[i].includes('='):
        [k, v] = tokens[i].split('=')
        env[k] = v
        i++
    else:
      args.push(tokens[i])
      i++

  config = loadMCPConfig(root)
  if config.mcpServers[name]:
    print "Server '#{name}' already exists. Use /mcp remove first."
    return true

  config.mcpServers[name] = { command, args, env }
  saveMCPConfig(root, config)

  // connect immediately via mcpBridge
  if mcpBridge and mcpBridge.connect:
    try:
      result = await mcpBridge.connect(name, config.mcpServers[name])
      toolCount = result.tools ? result.tools.length : 0
      print "Added and connected '#{name}' (#{toolCount} tools)"
    catch err:
      print "Added '#{name}' to config but connection failed: #{err.message}"
  else:
    print "Added '#{name}' to config. Reconnect with /mcp reload."

  return true


handleMCPRemove(root, tokens, mcpBridge):
  name = tokens[0]
  if not name:
    print "usage: /mcp remove <name>"
    return true

  config = loadMCPConfig(root)
  if not config.mcpServers[name]:
    print "Server '#{name}' not found in config."
    return true

  // disconnect first
  if mcpBridge and mcpBridge.disconnect:
    try:
      await mcpBridge.disconnect(name)
    catch err:
      // log but don't block removal

  delete config.mcpServers[name]
  saveMCPConfig(root, config)
  print "Removed '#{name}'"
  return true


handleMCPList(root, mcpBridge):
  config = loadMCPConfig(root)
  servers = config.mcpServers

  if Object.keys(servers).length == 0:
    print "No MCP servers configured."
    return true

  for name, serverConfig in servers:
    status = "configured"
    toolCount = 0

    if mcpBridge and mcpBridge.getStatus:
      try:
        s = mcpBridge.getStatus(name)
        status = s.connected ? "connected" : "disconnected"
        toolCount = s.toolCount || 0
      catch:
        status = "unknown"

    type = serverConfig.url ? "remote" : "stdio"
    print "  #{name} [#{type}] #{status} (#{toolCount} tools)"

  return true


handleMCPReload(root, mcpBridge):
  if not mcpBridge or not mcpBridge.disconnectAll or not mcpBridge.connect:
    print "MCP bridge not available."
    return true

  // disconnect all
  await mcpBridge.disconnectAll()

  // load config and reconnect each
  config = loadMCPConfig(root)
  connected = 0
  failed = 0

  for name, serverConfig in config.mcpServers:
    try:
      result = await mcpBridge.connect(name, serverConfig)
      toolCount = result.tools ? result.tools.length : 0
      print "  + #{name} (#{toolCount} tools)"
      connected++
    catch err:
      print "  x #{name}: #{err.message}"
      failed++

  print "Reloaded: #{connected} connected, #{failed} failed"
  return true


// === src/agent.js: handleSlashCommand update ===

handleSlashCommand(root, prompt, rl):
  [command, ...parts] = prompt.split(/\s+/)
  arg = parts.join(' ').trim()

  // ... existing commands ...

  if command == '/mcp':
    return handleMCPCommand(root, arg, mcpBridge)

  // ... rest unchanged ...

"Commands: /connect, /mcp [add|remove|list|reload], /models, /config, ..."


// === bin/self-improve-cli.js: main() additions ===

main():
  // ... after existing command blocks ...

  if command == 'mcp':
    [action, ...mcpRest] = rest

    if action == 'add':
      name = flags.name or mcpRest[0]
      cmd = flags.command
      if not name or not cmd:
        throw "usage: sicli mcp add <name> --command <cmd> [--args a1 a2 ...] [--env KEY=VAL]"

      mcpArgs = flags.args
        if typeof mcpArgs == 'string': mcpArgs = [mcpArgs]
        if not Array.isArray(mcpArgs): mcpArgs = []

      env = {}
      if flags.env:
        if typeof flags.env == 'string': parse KEY=VAL
        if Array.isArray(flags.env): parse each KEY=VAL

      config = loadMCPConfig(root)
      config.mcpServers[name] = { command: cmd, args: mcpArgs, env }
      saveMCPConfig(root, config)
      printJson({ ok: true, name, config: config.mcpServers[name] })

    else if action == 'remove':
      name = mcpRest[0]
      if not name:
        throw "usage: sicli mcp remove <name>"
      config = loadMCPConfig(root)
      if not config.mcpServers[name]:
        throw "Server '#{name}' not found"
      delete config.mcpServers[name]
      saveMCPConfig(root, config)
      printJson({ ok: true, removed: name })

    else if action == 'list':
      config = loadMCPConfig(root)
      servers = []
      for name, serverConfig in config.mcpServers:
        type = serverConfig.url ? 'remote' : 'stdio'
        servers.push({ name, type, command: serverConfig.command || serverConfig.url })
      printJson({ servers, count: servers.length })

    else:
      throw "unknown mcp action: #{action}. Use add, remove, or list."

//   sicli mcp remove <name>
//   sicli mcp list
```
